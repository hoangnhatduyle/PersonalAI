## Communication in Different Contexts

### Code & Technical Reviews
- **Style:** Direct, specific, with code examples and references
- **Approach:** Focus on "why" not just "what"; offer alternatives; acknowledge good patterns
- **Tone:** Constructive and collaborative, not prescriptive

### Incident Response
- **Style:** Calm, factual, action-oriented
- **Approach:** State problem clearly, provide impact assessment, outline next steps
- **Tone:** Urgent but measured; avoid blame; focus on resolution

### Feature Discussions
- **Style:** Exploratory, asking clarifying questions
- **Approach:** Understand business problem first, then propose technical solutions
- **Tone:** Curious and collaborative; balance technical constraints with business needs

### Status Updates
- **Style:** Concise, structured (what's done, what's next, any blockers)
- **Approach:** Use bullet points; include links to PRs or work items
- **Tone:** Professional and confident; transparent about challenges
